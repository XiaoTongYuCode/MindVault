# MindVault iOS 应用打包发布指南

## 前置准备

### 1. Apple 开发者账号
- 注册 [Apple Developer Program](https://developer.apple.com/programs/)（年费 $99）
- 确保账号已激活并可以访问 App Store Connect

### 2. 配置项目

#### 2.1 检查 Bundle Identifier
1. 在 Xcode 中打开 `MindVault.xcodeproj`
2. 选择项目 → Target: MindVault → General
3. 确认 **Bundle Identifier** 唯一（例如：`com.yourcompany.MindVault`）
4. 设置 **Version** 和 **Build** 号

#### 2.2 配置签名证书
1. 在 Xcode 中选择项目 → Target: MindVault → Signing & Capabilities
2. 勾选 **Automatically manage signing**
3. 选择你的 **Team**（Apple Developer 账号）
4. Xcode 会自动创建 Provisioning Profile

#### 2.3 配置 API Key
确保 `Config.xcconfig` 文件已配置且包含有效的 `OPENROUTER_API_KEY`：
```bash
# 如果还没有创建，从示例文件复制
cp Config.xcconfig.example Config.xcconfig
# 然后编辑 Config.xcconfig 填入你的 API key
```

#### 2.4 检查权限配置
确认 `Info.plist` 中已正确配置权限说明：
- 语音识别权限
- 麦克风权限

## 打包步骤

### 方法一：使用 Xcode GUI（推荐）

#### 1. Archive（归档）
1. 在 Xcode 顶部选择 **Any iOS Device**（不是模拟器）
2. 菜单栏：**Product** → **Archive**
3. 等待编译完成，会自动打开 Organizer 窗口

#### 2. 验证 Archive
1. 在 Organizer 窗口中选择刚创建的 Archive
2. 点击 **Validate App**
3. 选择分发方式：**App Store Connect**
4. 按照提示完成验证

#### 3. 分发到 App Store Connect
1. 在 Organizer 中选择 Archive
2. 点击 **Distribute App**
3. 选择 **App Store Connect**
4. 选择 **Upload**
5. 按照向导完成上传

### 方法二：使用命令行（适合 CI/CD）

```bash
# 1. 清理项目
xcodebuild clean -project MindVault.xcodeproj -scheme MindVault

# 2. Archive
xcodebuild archive \
  -project MindVault.xcodeproj \
  -scheme MindVault \
  -archivePath ./build/MindVault.xcarchive \
  -configuration Release \
  CODE_SIGN_IDENTITY="Apple Development" \
  DEVELOPMENT_TEAM="YOUR_TEAM_ID"

# 3. 导出 IPA（用于 App Store）
xcodebuild -exportArchive \
  -archivePath ./build/MindVault.xcarchive \
  -exportPath ./build/export \
  -exportOptionsPlist ExportOptions.plist
```

需要创建 `ExportOptions.plist`：
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>method</key>
    <string>app-store</string>
    <key>teamID</key>
    <string>YOUR_TEAM_ID</string>
</dict>
</plist>
```

## 发布到 App Store

### 1. 在 App Store Connect 创建应用
1. 登录 [App Store Connect](https://appstoreconnect.apple.com/)
2. 点击 **我的 App** → **+** → **新建 App**
3. 填写信息：
   - **平台**：iOS
   - **名称**：MindVault
   - **主要语言**：简体中文
   - **Bundle ID**：选择你配置的 Bundle Identifier
   - **SKU**：唯一标识符（例如：mindvault-001）

### 2. 准备应用信息
在 App Store Connect 中填写：
- **应用截图**（各种设备尺寸）
- **应用描述**
- **关键词**
- **隐私政策 URL**（如果应用需要）
- **分类**
- **年龄分级**

### 3. 提交审核
1. 上传 Archive 后，在 App Store Connect 中：
   - 选择 **版本** → **构建版本**
   - 选择你刚上传的构建版本
   - 填写版本信息
   - 点击 **提交以供审核**

### 4. 审核状态
- **等待审核**：通常 1-3 个工作日
- **审核中**：Apple 正在审核
- **已批准**：可以发布
- **被拒绝**：查看拒绝原因并修复

## 发布到 TestFlight（测试）

### 1. 内部测试
1. 在 App Store Connect 中进入你的应用
2. 选择 **TestFlight** 标签
3. 上传构建版本后，添加内部测试员（最多 100 人）
4. 测试员会收到邮件邀请

### 2. 外部测试
1. 在 TestFlight 中添加外部测试组
2. 填写测试信息（需要 Apple 审核，通常 24-48 小时）
3. 添加测试员（最多 10,000 人）

## 常见问题

### 1. 签名错误
- 检查开发者账号是否有效
- 确认 Bundle Identifier 唯一
- 重新生成 Provisioning Profile

### 2. 上传失败
- 检查网络连接
- 确认使用最新版本的 Xcode
- 查看具体错误信息

### 3. 审核被拒
- 仔细阅读拒绝原因
- 修复问题后重新提交
- 可以在 App Review Information 中添加说明

## 版本更新流程

1. 在 Xcode 中更新 **Version** 或 **Build** 号
2. 重新 Archive 和上传
3. 在 App Store Connect 中创建新版本
4. 提交审核

## 注意事项

- 准备完整的应用截图和描述
