## 1. 产品概述

### 1.1 产品定位

「私屿」是一款面向个人用户的情绪记录与分析应用。用户可以通过简洁优雅的界面记录每日日记，应用内置 AI 情感分析引擎（基于 OpenRouter LLM），自动识别日记中的情感倾向，并以可视化曲线追踪用户的情绪变化趋势。产品核心理念是"记录即疗愈"，帮助用户通过书写与回顾建立更好的自我认知。

### 1.2 目标用户

| 用户画像 | 描述 |
|---------|------|
| **核心用户** | 18-35 岁，有日记书写习惯或希望培养日记习惯的年轻人 |
| **关注心理健康者** | 希望通过记录和回顾了解自身情绪模式的用户 |
| **效率型用户** | 偏好简洁工具、不喜欢复杂社交功能的极简主义者 |

### 1.3 核心价值主张

> **用 AI 读懂你的情绪，让每一篇日记都有温度。**

产品通过三个核心能力为用户创造价值：第一，提供沉浸式的写作体验，让用户专注于表达；第二，通过 AI 自动分析每篇日记的情感倾向，无需用户手动标注；第三，以直观的情感曲线帮助用户发现自身情绪的周期性规律。

## 2. 功能需求

### 2.1 功能架构

应用采用三 Tab 底部导航架构，辅以模态页面实现核心交互流程。

| 模块 | 页面 | 优先级 | 状态 |
|------|------|--------|------|
| 首页 | 今日概览 + 最近日记 | P0 | 待开发 |
| 日记列表 | 全部日记按时间倒序 | P0 | 待开发 |
| 情感曲线 | 折线图 + 统计分布 | P0 | 待开发 |
| 写日记 | 全屏模态编辑器 | P0 | 待开发 |
| 日记详情 | 内容阅读 + 情感分析卡片 | P0 | 待开发 |
| 日记编辑 | 修改已有日记 | P1 | 待开发 |
| 搜索筛选 | 按关键词/情感标签筛选 | P1 | 待开发 |
| 深色模式切换 | 手动切换主题 | P2 | 待开发 |
| 数据导出 | 导出日记为 PDF/文本 | P2 | 待开发 |

### 2.2 首页 (Home Tab)

**功能描述：** 首页是用户打开应用后的第一个界面，提供今日概览和快速入口。

**功能要点：**

首页顶部展示基于当前时间的问候语（如"早上好"、"下午好"），配合副标题"记录你的心情与想法"，营造温暖的使用氛围。紧接着是"今日概览"卡片，展示当天已写日记数量和平均情感分数（附带对应的情绪 emoji）。页面中央放置醒目的"写日记"主操作按钮，采用紫色渐变填充，是整个首页的视觉焦点。下方展示最近 5 篇日记的预览卡片，每张卡片包含日期时间、标题、内容摘要（最多 2 行）以及情感标签。

**交互规则：**

| 操作 | 行为 |
|------|------|
| 点击"写日记"按钮 | 跳转到 Compose 模态页面 |
| 点击日记卡片 | 跳转到对应日记详情页 |
| 下拉 | 刷新数据 |

### 2.3 写日记 (Compose)

**功能描述：** 全屏模态页面，提供沉浸式的日记书写体验。

**功能要点：**

页面顶部为导航栏，左侧"取消"按钮、中间标题"写日记"、右侧"保存"按钮。导航栏下方依次是标题输入框（可选，placeholder 为"标题（可选）"）和正文编辑区域（placeholder 为"今天发生了什么？记录你的想法和感受..."）。正文区域支持多行输入，自动扩展高度，键盘弹起时页面自动适配。

**业务逻辑：**

用户点击"保存"后，系统执行以下流程：首先验证正文内容不为空；然后将日记数据（标题、内容、时间戳）写入 Core Data 本地存储；接着在后台异步调用 OpenRouter API 进行情感分析；分析完成后更新日记的 sentiment 字段。如果用户未填写标题，系统自动取正文前 20 个字符作为标题。

**异常处理：**

| 场景 | 处理方式 |
|------|---------|
| 正文为空点击保存 | 弹出提示"请输入日记内容" |
| 有未保存内容点击取消 | 弹出确认对话框"放弃编辑？" |
| API 调用失败 | 日记正常保存，情感分析标记为失败，不影响核心功能 |

### 2.4 日记列表 (Entries Tab)

**功能描述：** 展示用户所有日记条目，按创建时间倒序排列。

**功能要点：**

页面顶部为标题"日记列表"和"+ 新日记"快捷按钮。主体区域使用 SwiftUI List 高性能列表渲染所有日记条目。每个条目卡片展示：日期时间（左上）、情感标签 emoji + 文字徽章（右上，颜色区分积极/中性/消极）、日记标题（加粗，单行截断）、内容预览（最多 2 行）、以及 AI 生成的情感分析摘要（斜体引用样式）。列表为空时显示空状态插画和引导文案。

### 2.5 日记详情 (Entry Detail)

**功能描述：** 展示单篇日记的完整内容和 AI 情感分析结果。

**功能要点：**

页面顶部导航栏包含"返回"按钮（左）、标题"日记详情"（中）、"删除"按钮（右，红色）。内容区域依次展示：日期时间、日记标题（大号加粗）、完整正文内容（舒适的阅读排版，26px 行高）。正文下方是情感分析结果卡片，采用紫色渐变边框的毛玻璃效果，内部展示：情感 emoji（大号）、情感标签和分数、渐变色的分数位置指示条、以及 AI 生成的情感分析简述。

**删除逻辑：** 点击删除按钮后弹出二次确认对话框，确认后从 Core Data 中移除该条目并返回上一页。

### 2.6 情感曲线 (Mood Tab)

**功能描述：** 以可视化图表展示用户情感变化趋势和统计数据。

**功能要点：**

页面顶部为标题"情感曲线"和副标题"追踪你的情绪变化趋势"。下方是时间段选择器，提供"近 7 天"和"近 30 天"两个选项（药丸形按钮，选中态为紫色渐变填充）。

图表卡片内展示基于 Swift Charts 框架绘制的平滑贝塞尔曲线折线图。Y 轴范围为 -1 到 1，标注"积极"、"中性"、"消极"三个刻度；X 轴展示日期（MM-DD 格式）。数据点以彩色圆点标记（绿色=积极，黄色=中性，红色=消极），中性线（y=0）以虚线表示。同一天有多篇日记时，取平均分数作为该天的数据点。

图表下方展示三组统计卡片：平均情感分数（附带情感标签徽章）、情感分布（积极/中性/消极的数量和百分比，配合堆叠条形图）、以及所选时间段的日记总数。

## 3. 技术架构

### 3.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 框架 | SwiftUI + iOS SDK | 原生 iOS 开发 |
| 语言 | Swift 5.9+ | 类型安全 |
| 路由 | SwiftUI Navigation | 声明式导航 |
| 样式 | SwiftUI 原生样式系统 | 声明式 UI 构建 |
| 状态管理 | @State / @ObservableObject / @EnvironmentObject | SwiftUI 响应式状态 |
| 本地存储 | Core Data | 持久化日记数据 |
| 图表 | Swift Charts | 原生图表框架 |
| AI 服务 | OpenRouter API (URLSession) | LLM 情感分析 |

### 3.2 数据模型

```swift
struct DiaryEntry: Identifiable, Codable {
    let id: UUID              // UUID，唯一标识
    var title: String         // 日记标题
    var content: String       // 日记正文
    var createdAt: Date       // 创建时间
    var updatedAt: Date        // 更新时间
    var sentiment: Sentiment? // 情感分析结果
    
    struct Sentiment: Codable {
        let score: Double      // 情感分数 [-1, 1]
        let label: String     // "积极" | "中性" | "消极"
        let emoji: String     // 对应 emoji 表情
        let summary: String   // AI 生成的情感分析简述
    }
    
    var isAnalyzing: Bool = false  // 是否正在进行情感分析
}
```

### 3.3 AI 情感分析流程

情感分析采用异步后台处理模式，确保不阻塞用户的写作体验。具体流程如下：

1. 用户保存日记后，系统立即将日记数据写入本地存储，并将 `isAnalyzing` 标记为 `true`。
2. 系统构造 prompt，将日记内容发送至 OpenRouter API（默认使用 `google/gemini-2.0-flash-001` 模型）。
3. Prompt 要求 LLM 返回结构化 JSON，包含 score（-1 到 1 的浮点数）、label、emoji 和 summary 四个字段。
4. 收到响应后解析 JSON，更新日记的 sentiment 字段，将 `isAnalyzing` 设为 `false`。
5. 如果 API 调用失败（网络错误、Key 无效等），日记数据不受影响，仅情感分析结果缺失。

### 3.4 存储策略

应用采用纯本地存储方案，所有日记数据保存在设备的 Core Data 持久化存储中。Core Data 提供高效的数据管理和查询能力，支持自动增量更新。数据模型通过 Core Data 的 Entity 定义，支持关系型数据和复杂查询。这种方案的优势是零服务端依赖、隐私性强、离线可用、性能优异；局限是不支持跨设备同步。

## 4. 界面设计规范

### 4.1 设计语言

应用遵循 Apple Human Interface Guidelines (HIG)，追求简约、优雅、沉浸的设计风格。核心设计原则包括：

**简约克制：** 每个页面只保留必要元素，大量使用留白，让内容成为视觉主体。

**紫色渐变：** 主色调采用从深紫 `#7C3AED` 到浅紫 `#A78BFA` 的渐变，贯穿首页头部、主操作按钮、Tab 高亮等关键视觉元素，建立统一的品牌识别。

**毛玻璃质感：** 卡片和浮层采用 glassmorphism 效果（半透明背景 + 模糊 + 微边框），营造轻盈通透的层次感。

**微交互反馈：** 按钮按压时有 scale(0.97) 的缩放反馈，卡片按压时有 opacity 变化，配合 UIImpactFeedbackGenerator 提供触觉反馈。

### 4.2 色彩体系

| 色彩角色 | 亮色模式 | 暗色模式 | 用途 |
|---------|---------|---------|------|
| Primary | `#7C3AED` | `#A78BFA` | 主操作、Tab 高亮、渐变起点 |
| Primary Light | `#8B5CF6` | `#C4B5FD` | 渐变终点、次要强调 |
| Background | `#F8F7FF` | `#0F0F14` | 页面底色（微紫调） |
| Surface | `#FFFFFF` | `#1A1A24` | 卡片、浮层背景 |
| Foreground | `#1A1A2E` | `#E8E8F0` | 主文本 |
| Muted | `#6B7280` | `#9CA3AF` | 次要文本、placeholder |
| Border | `#E5E7EB` | `#2D2D3D` | 分割线、卡片边框 |
| Success | `#10B981` | `#34D399` | 积极情感 |
| Warning | `#F59E0B` | `#FBBF24` | 中性情感 |
| Error | `#EF4444` | `#F87171` | 消极情感 |

## 5. 用户流程

### 5.1 核心流程：写日记并查看情感分析

```
打开应用 → 首页（今日概览）
    ↓
点击"写日记"按钮
    ↓
进入 Compose 页面 → 输入标题（可选）→ 输入正文
    ↓
点击"保存"
    ↓
日记保存成功 → 返回首页（日记卡片出现，显示分析中...）
    ↓
后台 AI 分析完成 → 日记卡片更新情感标签和 emoji
    ↓
点击日记卡片 → 进入详情页 → 查看完整内容和情感分析结果
```

### 5.2 核心流程：查看情感趋势

```
切换到"情感"Tab
    ↓
默认展示近 7 天情感曲线
    ↓
查看折线图趋势 → 识别情绪高点和低点
    ↓
查看统计数据 → 了解积极/中性/消极分布
    ↓
切换到"近 30 天"→ 查看更长周期的趋势
```

## 6. 非功能需求

### 6.1 性能要求

| 指标 | 目标值 |
|------|--------|
| 应用启动时间 | < 2 秒 |
| 日记保存响应 | < 500ms（本地存储） |
| 情感分析延迟 | < 5 秒（依赖网络） |
| 列表滚动帧率 | 60 FPS |
| 内存占用 | < 150MB |

### 6.2 隐私与安全

所有日记数据存储在用户设备本地（Core Data），不上传至任何服务器。情感分析时仅将日记文本内容发送至 OpenRouter API 进行处理，不传输用户身份信息。OpenRouter API Key 通过 Info.plist 或环境变量注入，不硬编码在客户端代码中。

## 8. 附录

### 8.1 术语表

| 术语 | 定义 |
|------|------|
| 情感分数 (Sentiment Score) | 范围 [-1, 1] 的浮点数，-1 表示极度消极，0 表示中性，1 表示极度积极 |

| OpenRouter | 第三方 LLM API 聚合平台，提供多种大语言模型的统一调用接口 |
| Core Data | iOS 的原生持久化存储框架，提供对象关系映射和高效数据管理 |
| Glassmorphism | 毛玻璃设计风格，通过半透明背景和模糊效果营造层次感 |
